--> While the specific properties of an event object can vary depending on the type of event and the browser, 
    here's a general overview of the most common ones:

1) Core Properties:

  - type: The type of event (e.g., "click", "keydown", "mouseover").
  - target: The element that triggered the event.
  - currentTarget: The element on which the event listener is attached.
  - timeStamp: The time at which the event occurred (in milliseconds).
  - isTrusted: Whether the event was generated by a user action or by the script itself.

2) Mouse Event Properties:

  - clientX: The X coordinate of the mouse pointer relative to the viewport.
  - clientY: The Y coordinate of the mouse pointer relative to the viewport.
  - pageX: The X coordinate of the mouse pointer relative to the document.
  - pageY: The Y coordinate of the mouse pointer relative to the document.
  - screenX: The X coordinate of the mouse pointer relative to the screen.
  - screenY: The Y coordinate of the mouse pointer relative to the screen.
  - button: The mouse button that was pressed (0 for left, 1 for middle, 2 for right).

3) Keyboard Event Properties:

  - keyCode: The key code of the key that was pressed.
  - charCode: The character code of the key that was pressed.
  - key: The key that was pressed (e.g., "a", "Enter", "ArrowUp").

4) Form Event Properties:

  - value: The value of the form element.
  - checked: Whether a checkbox or radio button is checked.
  - selectedIndex: The index of the selected option in a select element.

5) Additional Properties:

--> Depending on the specific event type, there may be additional properties available. For example, 
    the UIEvent interface provides properties like detail, view, and relatedTarget, 
    which can be useful for touch events and other UI-related interactions.

--> Remember: The specific properties available will vary depending on the browser and the type of event. 
    It's always best to consult the browser's documentation for the most accurate information.
